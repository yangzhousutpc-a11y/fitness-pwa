# 健身 PWA 全量 UI/交互优化规格

版本：2026-06-26

状态：设计规格，尚未进入代码实现

## 1. 目标

当前产品已经具备训练计划、动作库、逐组记录、休息计时、历史记录和数据库同步能力。本轮只优化 UI 和交互效果，目标是让产品更像一个高频使用的训练工具：

- 打开首页后，用户能更快进入训练。
- 进入计划页后，用户能更快选择训练日。
- 训练中，记录重量和次数不被大图、说明文字、导航和多余控件打断。
- 训练后，历史页能更清楚地反馈训练结果。

## 2. 范围

本规格覆盖当前产品已有的所有主要 UI 表面：

- 全局框架和底部导航
- 数据同步和访问密钥提示
- 首页
- 内置计划详情页
- 我的计划库
- 自定义计划编辑页
- 动作库
- 训练记录页
- 历史页

## 3. 不做

- 不新增产品功能。
- 不新增页面。
- 不改数据库。
- 不改 API。
- 不改训练计划内容。
- 不新增账号体系、社交、分享、订阅、支付、AI 推荐。
- 不新增复杂数据图表。
- 不改变现有页面路由和核心业务流程。

## 4. 设计方向

方向：训练优先型。

这个产品不是内容站，也不是营销页，而是训练时会被反复打开的手机工具。界面应该服务三个高频问题：

- 今天从哪里开始练？
- 当前正在录哪个动作、哪一组？
- 刚才完成后下一步做什么？

视觉语气：

- 深色、克制、力量训练工具感。
- 保留绿色作为主状态色，但减少大面积荧光和过多发光边框。
- 文字层级更清楚，少用解释性长文。
- 卡片减少嵌套，避免“卡片里面套卡片”的厚重感。

## 5. 全局 UI 规则

### 5.1 色彩

主绿色只用于：

- 主按钮
- 当前组
- 已完成状态
- 休息计时器
- 关键反馈数字

灰白色用于：

- 普通标题
- 次级说明
- 肌群、器械、时间等辅助信息

红色只用于：

- 删除
- 同步失败
- 不可恢复或需要注意的错误状态

避免：

- 每个卡片都有绿色描边。
- 每个数字都用绿色。
- 大面积高饱和背景。

### 5.2 字体层级

建议层级：

- 页面标题：24-30px，仅用于当前页面主标题。
- 区块标题：16-18px，用于“本周概览”“动作列表”等。
- 卡片主标题：16-20px，用于计划名、动作名。
- 表格/输入标签：12-13px。
- 辅助说明：12-14px。

训练页输入区域中，字体不应过大。训练中需要的是快速扫读，不是展示型大标题。

### 5.3 间距和圆角

- 页面左右边距保持 20-24px。
- 训练页可适当收紧到 16-20px。
- 卡片圆角不超过 8-12px。
- 按钮高度不低于 40px。
- 主要触控目标不低于 40px。

### 5.4 动效

动效只用于帮助用户理解状态变化：

- 展开/折叠：150-200ms。
- 当前组高亮：150ms 以内。
- 底部导航退出输入模式：150-200ms。
- 休息计时器出现：轻微上滑或淡入。

避免：

- 弹跳动画。
- 长时间过渡。
- 会影响记录效率的装饰动画。

### 5.5 底部导航

当前底部导航保留三个入口：

- 计划
- 动作库
- 历史

训练中输入重量/次数时：

- 底部导航应退出视口或降低干扰。
- 不遮挡输入框。
- 不造成页面明显跳动。

非训练页：

- 底部导航固定可见。
- 当前 tab 状态清楚，但不要用过亮的大面积背景。

## 6. 全局状态

### 6.1 数据同步中

当前已有“正在同步数据库”提示。优化方向：

- 位置靠近顶部，但高度尽量紧凑。
- 使用中性颜色，不要像错误状态。
- 文案保持短句：`正在同步数据库...`

验收：

- 不遮挡首页主操作。
- 不导致首屏布局剧烈变化。

### 6.2 同步失败或访问密钥错误

当前已有访问密钥输入。优化方向：

- 错误提示与输入框归为同一个紧凑状态条。
- 主按钮文案为 `保存并重试`。
- `重试` 做次级按钮。
- 错误信息不要占据大面积首屏。

验收：

- 手机首屏仍能看见页面主内容的一部分。
- 输入框、保存按钮、重试按钮不换行挤压到不可点。

## 7. 首页 UI 规格

### 7.1 首页目标

首页的第一任务是让用户开始训练，而不是同时展示所有模块。

### 7.2 当前问题

当前首页中：

- 计划入口、周概览、个人最好成绩、最近记录权重接近。
- 用户第一次打开时，需要自己判断哪个入口最重要。
- 统计模块偏早出现，训练前会分散注意力。

### 7.3 推荐结构

首屏顺序：

1. 主训练入口区
2. 计划入口区
3. 最近记录
4. 本周概览
5. 个人最好成绩

说明：如果不改数据逻辑，可以使用当前已有内置计划入口作为主训练入口，只调整视觉层级。

### 7.4 主训练入口区

视觉：

- 占据首页第一屏的核心位置。
- 一个主卡片，不并排塞多个等权重卡片。
- 主按钮清楚，例如 `进入名师计划` 或 `开始训练`。

内容：

- 计划名
- 教练名
- 训练天数
- 主操作按钮

不建议：

- 同屏放 3 个同样醒目的大卡片。
- 把统计数字放在主入口上方。

验收：

- 390x844 手机首屏无需滚动即可看见主训练入口。
- 主操作按钮在 3 秒内可被识别。
- 统计模块不抢主按钮视觉焦点。

### 7.5 计划入口区

视觉：

- 内置计划和自定义计划可以继续存在。
- 非主推计划使用更轻的列表或次级卡片。
- 自定义计划入口不应比主训练入口更抢眼。

验收：

- 用户仍能找到“我的计划”。
- 入口之间主次明确。

### 7.6 最近记录

视觉：

- 最近记录应比统计更接近训练入口。
- 使用单行或紧凑卡片。

内容：

- 训练日名称
- 日期
- 完成组数或简单摘要，如已有数据支持则展示；不新增计算也可以只保留现有信息。

验收：

- 用户能快速看到上次训练是什么。
- 不占据过多首屏空间。

### 7.7 本周概览和个人最好成绩

视觉：

- 放在首页中下部。
- 数字可保留，但弱于主训练入口。
- PR 列表不需要首屏完整展示。

验收：

- 统计信息清楚但不喧宾夺主。
- 没有训练数据时，空状态轻量，不像错误。

## 8. 内置计划详情页 UI 规格

### 8.1 页面目标

计划详情页的任务是帮助用户选择训练日并开始训练。

### 8.2 当前问题

当前计划页中：

- 计划说明占据较多注意力。
- Day 卡片、动作列表、名师要点同时出现，信息量较大。
- `开始训练` 不够像当前 Day 的主按钮。

### 8.3 推荐结构

页面顺序：

1. 计划摘要
2. Day 列表
3. 展开的 Day 内容
4. 当前 Day 的开始训练按钮
5. 视频入口

### 8.4 计划摘要

视觉：

- 压缩高度。
- 只保留教练名、计划名、一句话描述。
- 原视频入口做次级链接。

验收：

- 首屏能同时看到计划摘要和至少一个 Day 卡片。
- 计划摘要不形成大段阅读压力。

### 8.5 Day 卡片

视觉：

- Day 卡片比动作卡片更高一层。
- 展开态和折叠态边界清楚。
- Day 名称、肌群、动作数量保持可扫读。

交互：

- 点击 Day 标题展开/折叠。
- 展开后不自动挤掉当前上下文。

验收：

- 390x844 首屏能看到 Day 标题和开始训练路径。
- 展开/折叠状态通过箭头和视觉背景都能识别。

### 8.6 动作列表

视觉：

- 默认像清单，而不是文章。
- 动作名为主，肌群/器械为辅。
- 组数进度在训练页重要，计划页可以弱化。

名师要点：

- 默认降低视觉权重。
- 图片不应在计划页首屏大面积铺开。
- 可作为展开内容存在。

验收：

- 用户能快速扫到这一天有哪些动作。
- 名师图文不会压过“开始训练”。

### 8.7 开始训练按钮

视觉：

- 与当前展开 Day 绑定。
- 使用主绿色。
- 不要漂在过远的位置。

验收：

- 展开 Day 后，不滚动太多就能看到 `开始训练`。
- 主按钮比视频入口更醒目。

## 9. 我的计划库 UI 规格

### 9.1 页面目标

帮助用户管理自己的自定义计划。

### 9.2 推荐结构

1. 新建计划按钮
2. 我的计划列表
3. 空状态

### 9.3 新建按钮

视觉：

- 保留主按钮，但避免占据过高空间。
- 文案简洁：`新建自定义计划`。

验收：

- 空列表时，用户能明确知道从哪里创建。
- 有计划时，新建按钮仍可见但不压过计划列表。

### 9.4 计划卡片

视觉：

- 标题、动作数量、肌群标签、操作按钮分区清楚。
- 编辑、进入、删除三类操作视觉权重不同。
- 删除保持危险色，但不要和主操作同权重。

验收：

- 计划标题长时不溢出。
- 三个操作按钮在手机上不拥挤。

## 10. 自定义计划编辑页 UI 规格

### 10.1 页面目标

让用户快速搭建一个可训练的动作列表。

### 10.2 推荐结构

1. 计划名称和描述
2. 当前动作列表
3. 添加动作区域
4. 开始训练按钮

### 10.3 计划名称和描述

视觉：

- 名称输入更突出。
- 描述输入弱化，不占过高空间。

验收：

- 用户进入后优先看到计划名和动作列表。
- 描述为空时不显得页面未完成。

### 10.4 动作列表

视觉：

- 使用紧凑列表。
- 序号、动作名、删除按钮对齐稳定。
- 空状态给轻量提示。

验收：

- 390x844 下能看到多个动作。
- 删除按钮不误触主操作。

### 10.5 添加动作区域

视觉：

- 搜索框和筛选标签保持紧凑。
- 添加结果列表不要像大卡片墙，优先可扫读。

验收：

- 搜索、筛选、添加三步在同一屏附近完成。
- 筛选标签不横向溢出。

## 11. 动作库 UI 规格

### 11.1 页面目标

让用户查动作、看肌群和动作说明。

### 11.2 推荐结构

1. 搜索框
2. 肌群筛选
3. 动作列表
4. 动作详情或提示内容

### 11.3 搜索和筛选

视觉：

- 搜索框固定在列表上方。
- 筛选标签做紧凑 pill。
- 当前筛选清楚，但不要过亮。

验收：

- 手机下筛选不产生横向滚动。
- 搜索框聚焦时不遮挡列表第一项。

### 11.4 动作列表

视觉：

- 动作名优先。
- 肌群/器械作为辅助。
- 卡片高度统一。

验收：

- 用户能快速扫过 8-10 个动作。
- 动作名较长时不撑破布局。

## 12. 训练记录页 UI 规格

### 12.1 页面目标

训练页是最高优先级页面。它只服务一件事：训练中快速记录，不打断节奏。

### 12.2 当前问题

当前训练页已经有逐组输入、休息计时、输入聚焦模式和完成状态。但仍需要继续降噪：

- 名师图文容易占据训练首屏。
- 顶部、休息条、动作卡片、底部导航容易同时争夺注意力。
- 记录区需要更强的当前组和下一组视觉层级。

### 12.3 推荐结构

训练页视觉顺序：

1. 顶部简洁标题
2. 训练进度条或进度摘要
3. 休息计时器
4. 当前动作标题
5. 逐组记录表
6. 名师要点
7. 添加/删除组、备注

说明：这不是新增功能，只是调整视觉权重和位置。

### 12.4 顶部标题

普通状态：

- 显示 Day 名称和肌群。
- 返回按钮保留。

输入聚焦状态：

- 压缩到一行。
- 隐藏次级说明。
- 返回按钮缩小但仍可点。

验收：

- 输入时顶部不占用大量高度。
- 不遮挡当前输入行。

### 12.5 训练进度摘要

视觉：

- `0/25 组完成` 保留，但不需要过大。
- `完成` 按钮保留主操作感。
- 可使用紧凑条而不是厚重卡片。

验收：

- 用户能看到总进度。
- 进度区不挤压当前动作记录。

### 12.6 休息计时器

视觉：

- 作为紧凑辅助条。
- 左侧倒计时最醒目。
- 中间展示下一组提示。
- 右侧按钮减少视觉噪音。

交互：

- 完成一组后出现。
- 输入下一组时不应遮挡输入区域。
- 跳过按钮存在但弱于倒计时。

验收：

- 完成一组后，能看到休息剩余时间和下一组。
- 手机首屏中休息条不遮挡当前组输入。
- 计时器不是弹窗感，不打断训练。

### 12.7 当前动作标题

视觉：

- 动作标题吸顶时应有边界或轻阴影。
- 动作名、肌群/器械、完成进度保持可读。
- 折叠箭头触控区域足够大。

验收：

- 滚动到动作底部时仍能快速折叠当前动作。
- 吸顶标题不遮挡输入行。

### 12.8 名师要点

普通训练状态：

- 默认视觉权重降低。
- 图片不应在首屏占据最大面积。
- 文字说明以短句清单为主。

输入聚焦状态：

- 可以降低透明度或收缩高度。
- 不要通过突然 `display: none` 造成页面跳动。

验收：

- 打开训练页首屏至少能看到当前动作和部分记录区。
- 名师要点可读，但不是主视觉。

### 12.9 逐组记录表

视觉：

- 表头清楚：组、重量、次数、完成。
- 当前组用轻绿色背景。
- 已完成组用更克制的绿色，不要整行过亮。
- 未完成普通组保持低调。

交互：

- 点击重量/次数进入输入聚焦模式。
- 完成一组后，下一组被定位和高亮。
- 不自动弹出下一组键盘，避免打断休息。

验收：

- 390x844 手机首屏至少看到 2-3 组输入行。
- 完成组、当前组、未完成组三种状态一眼能分辨。
- 输入框不横向挤压，数字不会溢出。

### 12.10 添加/删除组和备注

视觉：

- 放在记录表之后。
- 比记录表弱，不要抢主路径。
- 备注框高度适中。

验收：

- 用户训练中不容易误触添加/删除组。
- 备注不挤压记录表。

## 13. 历史页 UI 规格

### 13.1 页面目标

历史页用于训练后回看，而不是训练中操作。它应该比训练页更安静。

### 13.2 推荐结构

1. 最近训练摘要
2. 日历
3. 所选日期训练摘要
4. 个人最好成绩
5. 动作进步记录

如果当前页面已有类似模块，只调整视觉层级，不新增新分析能力。

### 13.3 最近训练摘要

视觉：

- 放在历史页靠前位置。
- 显示最近一次训练名称、日期、完成组数。

验收：

- 用户进入历史页后能快速找到最近训练。
- 摘要不需要大面积卡片。

### 13.4 日历

视觉：

- 当前选中日期清楚。
- 有训练的日期做轻量标记。
- 无训练日期不应像错误状态。

验收：

- 点击日期后，所选日期状态明确。
- 小屏无横向滚动。

### 13.5 所选日期训练摘要

视觉：

- 训练名、完成组数、动作数分层。
- 无训练时使用轻量空状态。

验收：

- 用户能判断当天是否训练。
- 有训练时摘要信息比空状态更醒目。

### 13.6 PR 和进步信息

视觉：

- PR 数字可以保留绿色，但不要所有信息都绿色。
- 动作名和最佳重量层级清楚。
- 图表或进步记录若已有，视觉上作为回看内容，不抢日历。

验收：

- 用户能看出哪些动作有最好成绩。
- 统计卡片在手机上不拥挤。

## 14. 页面优先级

### P0 - 训练页

原因：训练页是最高频、最容易因 UI 干扰影响体验的页面。

验收：

- 输入重量/次数时页面稳定。
- 完成一组后，完成组和下一组状态清楚。
- 休息计时器和下一组提示不遮挡记录。
- 名师要点不抢记录区主视觉。

### P1 - 首页

原因：首页决定用户进入训练的速度。

验收：

- 主训练入口成为第一视觉焦点。
- 统计信息后移并降权。
- 首页首屏主次明确。

### P1 - 计划详情页

原因：计划页决定用户能否快速选择训练日并开始训练。

验收：

- Day 卡片更像选择器。
- 开始训练按钮与当前 Day 绑定明确。
- 动作说明不压过开始路径。

### P2 - 历史页

原因：历史页影响训练反馈感，但不应早于训练效率。

验收：

- 最近训练和所选日期摘要清楚。
- 日历状态明确。
- PR 和统计信息安静可读。

### P3 - 全局视觉统一

原因：统一后产品会更像完整工具，而不是多个局部样式拼接。

验收：

- 主绿色使用有纪律。
- 卡片、按钮、表格、标签圆角和间距统一。
- 展开/折叠、输入聚焦、休息条出现动效一致。

## 15. 响应式验收

手机宽度优先：

- 390x844：主验收尺寸。
- 375x812：兼容较窄手机。
- 430x932：兼容大屏手机。

检查项：

- 无横向滚动。
- 文字不溢出按钮。
- 输入框不被底部导航或键盘遮挡。
- 卡片之间不出现异常重叠。
- 吸顶标题不遮挡当前输入区域。

## 16. 空状态和错误状态

### 首页无训练记录

文案应轻量：

- `完成一次训练后，这里会显示最近记录。`

避免：

- 使用错误色。
- 大面积空白卡片。

### 我的计划为空

文案应指向已有操作：

- `还没有自定义计划，点上方新建。`

避免：

- 新增引导流程。

### 历史日期无训练

文案：

- `当天没有训练记录。`

视觉：

- 中性灰。
- 不使用警告色。

### 数据同步失败

视觉：

- 使用错误色，但高度紧凑。
- 输入 token 和重试操作清楚。

## 17. 可访问性和触控

- 所有按钮触控高度不低于 40px。
- 折叠箭头不应只有小图标可点，整行标题可点。
- 输入框 label 保持可访问性。
- 完成按钮需要通过形状和文字/符号同时表达状态，不能只靠颜色。
- 当前组和完成组不能只靠颜色区分，也应有边框、图标或明暗差。

## 18. 实施边界

后续如果进入开发，只允许改：

- CSS 布局
- 视觉层级
- 现有组件的排列
- 现有交互的动效和状态表现
- 文案短句

不允许改：

- 数据结构
- API
- 数据库
- 新页面
- 新业务流程
- 新统计口径
- 新训练推荐逻辑

## 19. 建议的实现顺序

1. 训练页记录区降噪。
2. 训练页休息计时器紧凑化。
3. 首页主训练入口重排。
4. 计划详情页 Day 卡片和开始训练入口重排。
5. 历史页最近训练和日历层级优化。
6. 全局颜色、间距、圆角、动效统一。

## 20. 设计验收清单

实现后逐项检查：

- 首页首屏能明确看到主训练入口。
- 计划页首屏能看到 Day 选择和开始训练路径。
- 训练页首屏能看到当前动作和至少 2-3 组输入行。
- 输入重量或次数时，页面不明显跳动。
- 底部导航不遮挡输入框。
- 完成一组后，完成组、下一组、休息计时器状态都清楚。
- 名师要点不抢记录区主视觉。
- 历史页能快速找到最近训练。
- 手机 375、390、430 宽度均无横向滚动。
- 所有主要按钮文字不溢出。
- 空状态不使用错误视觉。
